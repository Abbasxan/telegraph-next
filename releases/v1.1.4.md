## 🌟 Overview

**Telegraph-Next v1.1.4** is a massive leap forward in stability and reliability. We tackled the two biggest issues developers face when using Telegra.ph: **ISP SNI Blocks** and **Unstable Image Uploads**. 

With this release, `telegraph-next` becomes the most resilient Telegraph API wrapper available, ensuring your bots and services remain online regardless of regional restrictions or Telegra.ph server outages.

---

## 🔥 Key Features

### 🛡️ 1. Native DPI Bypass (Anti-Block)
Many ISPs block Telegra.ph via DNS or SNI. `telegraph-next` now natively routes connections directly through Telegram's core IP addresses (`149.154.164.13`). 
* **Zero Config**: No proxies or VPNs required.
* **Instant Speed**: Bypasses DNS resolution entirely.

### 🖼️ 2. Universal Image Uploads (Catbox.moe)
The official `telegra.ph/upload` endpoint is globally unstable and often throws `Unknown error`. We have completely overhauled the `upload_file()` method:
* Natively uses **Catbox.moe** API as an ultra-reliable fallback.
* Guarantees 100% upload success rates.
* Returns a fully qualified `UploadedFile` object with a permanent media link.

### 🧩 3. Zero Conflicts (Drop-in Replacement)
To prevent "Shadowing" conflicts with the abandoned official library, the internal package has been renamed to `telegraph_next`. You can now safely install it alongside legacy code.

## 🛠️ How to Use

### 1. Creating a Post (Classic Text)
```python
import asyncio
from telegraph_next import Telegraph

async def main():
    async with Telegraph() as tg:
        await tg.create_account(short_name='Abbasxan')
        
        page = await tg.create_page(
            title='Anti-DPI Test',
            content_html='<h1>Hello World</h1><p>Bypassing blocks effortlessly!</p>'
        )
        print(f"Page URL: {page.url}")

asyncio.run(main())
```

### 2. Uploading Media (Catbox Fallback)
```python
import asyncio
from telegraph_next import Telegraph

async def upload_demo():
    async with Telegraph() as tg:
        # Simply pass the file path
        uploaded_media = await tg.upload_file("photo.jpg")
        
        # The library returns the full accessible URL
        print(f"Media successfully uploaded to: {uploaded_media.src}")

asyncio.run(upload_demo())
```
